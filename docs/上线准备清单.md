## 上线前必须解决的问题

### P0 — 必须解决（不解决不能上线）

**1. 测试跑不通**
- 测试全部超时，可能是因为E2E测试调用了真实LLM API
- 需要：纯单元测试不依赖外部API，E2E测试需要mock或标记跳过
- 目标：`pytest tests/ -v` 能在2分钟内跑完全部通过

**2. 安全：API Key泄露风险**
- .env里有真实的DeepSeek和MiniMax API Key
- .gitignore必须排除.env
- .env.example里应该是占位符不是真实key
- 检查代码里有没有硬编码的key

**3. 错误处理不完善**
- LLM API调用失败时（超时、限流、key无效）应该有优雅降级
- WebSocket断连后资源清理
- 全局异常捕获，不能让服务直接崩溃

**4. 输入校验**
- 用户输入的idea不能是空字符串
- WebSocket消息格式校验
- 文件名防路径穿越（download接口）

### P1 — 应该解决（上线后体验差）

**5. 健康检查接口**
- 已有 `/health`（src/utils/health.py），需要确认和路由联通
- 容器编排需要这个接口判断服务是否正常

**6. 日志系统**
- 已有结构化日志（src/utils/logger.py），需要确认全局使用
- 日志级别：开发DEBUG、生产INFO
- 请求日志：每次WebSocket连接/断连记录

**7. CORS配置**
- 如果前后端分离部署，需要配置CORS
- 当前前后端同源（都从FastAPI serve），暂时不需要

**8. 超时控制**
- 单个Agent调用LLM的超时时间（建议30秒）
- 整个流水线的超时时间（建议5分钟）
- WebSocket心跳保活

**9. 并发限制**
- 同时运行的流水线数量上限（防止API Key额度耗尽）
- 每个IP的连接数限制

### P2 — 锦上添花

**10. README完善**
- 项目介绍、截图、快速开始、API文档链接
- 部署指南（Docker一键启动）

**11. .env.example完善**
- 所有环境变量都有注释说明
- 必填项和可选项标注清楚

**12. Docker镜像优化**
- .dockerignore排除不需要的文件
- 多阶段构建减小镜像体积
- 不要把.venv和__pycache__打包进去

**13. 启动脚本**
- start.sh已存在，确认Linux下能用
- 生产环境去掉--reload

## 验收标准

```bash
# 1. 测试通过
pytest tests/ -v  # 全绿，2分钟内完成

# 2. 代码质量
ruff check src/ tests/  # 无报错

# 3. 服务启动
python -m uvicorn src.web.server:app --host 0.0.0.0 --port 8080
# 访问 http://localhost:8080 能看到界面
# 浏览器输入想法能走完全流程

# 4. 健康检查
curl http://localhost:8080/health  # 返回200

# 5. Docker
docker build -t ai-dev-platform .
docker run -p 8080:8080 --env-file .env ai-dev-platform
# 能正常访问

# 6. 安全
# .env不在git中
# .env.example只有占位符
# 代码中无硬编码key
```
